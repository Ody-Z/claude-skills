# 小红书笔记创作

Ody 的个人 IP 是"AI 全栈 Builder，把点子变成现实的人"。不是博主，是做个人品牌，注重输出质量。内容定位是认知深度和信息密度，不是情绪刺激和流量套路。

严格按以下 4 阶段顺序执行。Phase 1 和 Phase 2 完成后，等 Ody 确认再往下走。Phase 3 和 Phase 4 连续执行，中途不要停下来询问，唯一的例外是 Phase 4 的中英文对照表需要 Ody 确认后再统一修改。

## Phase 1: 选题与切入角度

动笔之前先想清楚：

1. **目标观众** — 明确这篇内容写给谁看：
   - AI 从业者/技术人员
   - 对 AI 感兴趣但不了解技术的普通人
   - 创业者/投资人
   - 特定行业人群（如设计师、律师、产品经理等）
   - 用具体画像描述（如"工作 3-5 年、对 AI 好奇但没深入使用过的产品经理"）

2. **核心论点** — 用一句话说清楚这篇要表达什么。论点要有明确立场，不能模棱两可。好的论点自带认知冲突：让人觉得"我以为我知道，但好像没那么简单"

3. **认知冲突点** — 这篇内容制造的认知张力是什么？两个权威来源的对立判断、旧行业类比解释新行业、反常识的具体案例——要让读者停下来想一想

4. **信息支撑** — 素材来源是什么？偏好权威信息源（a16z、Howard Marks、Paul Graham、行业创始人访谈、技术论文、播客深度对话）。Ody 几乎不写纯个人感悟，每篇都有外部信息支撑
   - **时效性要求**：优先使用 2026 年最新数据、案例和行业动态。如果引用历史数据或案例，必须说明时间背景

5. **实用价值** — 读者看完能带走什么？一个框架、一个新视角、一条判断逻辑、还是一个被忽视的事实？

**呈交给 Ody 确认：**
- 目标观众（具体画像）
- 核心论点（一句话）
- 认知冲突点
- 信息来源/素材（标注时间，优先 2026 年最新）
- 建议标题（2-3 个备选，风格见 references/style-guide.md 标题策略）
- 预估篇幅：短帖（300-800 字）还是长文（1000-2500 字）
- 建议封面方向（白底黑字为主，信息密度高，直接陈述观点或制造认知冲突）

## Phase 2: 结构与大纲

角度确认后：

1. **结构选型** — 根据内容选择结构模式：
   - **"N 点并列洞察"**：适合行业分析帖，每个分论点是"观点 → 案例 → 类比 → 短句收尾"，小标题用直接陈述而非修辞句式
   - **"线性论证"**：适合长文，一层推一层，"现象观察 → 路径梳理 → 战略分析 → 开放收尾"
2. **小标题** — 每个段落有清晰的信息点，小标题是直接陈述，不玩修辞
3. **素材储备** — 每个要点至少配一个：权威引用、真实数据、对比案例、跨行业类比
4. **信息核实** — 定稿大纲前，上网搜索核实（优先使用 2026 年最新信息）：
   - 提到的产品名称、版本、功能是否最新（2026 年 2 月的最新状态）
   - 数据和统计是否准确、来源可靠、时效性如何（优先 2025-2026 年数据）
   - 案例和故事是否属实、时间线是否正确
   - 引用的人物观点是否准确归属、是否为最新表态
   - **避免使用过时案例**：如果某个产品/公司/趋势在 2026 年已经过时或被取代，必须更新
5. **配图规划** — 封面图 + 正文配图（流程图、对比图、框架图等）：
   - 封面：白底黑字，信息密度高，直接陈述观点或制造认知冲突
   - 正文配图用 `[配图: 描述]` 标注，搜索参考图片提供链接
6. **开头** — 第一句话直接进入，零铺垫。用反常识事实、具体数据、或一个让人意外的观察切入
7. **结尾** — 可以用开放性问题收尾引发思考，但全文必须有明确的判断和立场。不做"两边都有道理"的骑墙派

呈交完整大纲（含素材和配图规划），等 Ody 确认。

## Phase 3 + 4: 撰写初稿与审稿（连续执行，不中断）

Phase 3 初稿写完后，**直接进入 Phase 4 审稿，不停下来询问 Ody**。唯一需要暂停等待确认的是：Phase 4 第 5 步的中英文对照表。

## Phase 3: 撰写初稿

详细写作规范见 references/style-guide.md。

**核心要求：**
- Ody 的声音：谦逊但有立场的观察者，不是导师。用"我发现""最近在看/在研究"切入，但一定会给出自己的明确判断
- 段落极短：1-3 句/段，短句独立成段制造节奏感
- **换行规则**：正文内所有句子之间只用单换行（\n），全文正文里不出现空行（\n\n）。唯一例外：### 小标题前保留一个空行（Markdown 渲染需要）。[配图:] 标注行也遵守同一规则，前后都是单换行。
- **粗体短标题**：段落内的粗体短标题（如 **谁最危险**、**关键原则**）不加句号
- 重点加粗，让扫读也能 get 核心判断
- 每个观点必须有信息支撑（数据、案例、权威引用），不空喊口号
- **信息密度优先**：少文学性多信息量，少感受多判断
- **时效性优先**：所有数据、案例、产品功能必须是 2026 年最新状态。如果引用历史案例，明确标注时间（如"2023 年 GPT-4 发布时..."）
- 用 `[配图: 描述]` 标注配图位置
- 结尾可以用开放性问题引发讨论，但全文立场必须清晰
- **简介引言与正文同步生成**：简介引言不是正文的摘录，而是用 1-3 句话概括文章的中心思考，让读者在点进来之前就能判断这篇值不值得读。写完正文后立即写简介引言，一起输出到文件

**Ody 的写法特征：**
- 呈现对立判断但不急于站队（Howard Marks vs Gavin Baker）
- 用旧行业类比新行业（硬盘行业预言 AI 竞赛）
- 反常识的具体案例做论据（Notion 联创是最强程序员但不再写代码）
- 术语本土化（Slack → 飞书），关键英文术语保留加中文标注（MCP/Model Context Protocol，模型上下文协议）
- 偶尔自嘲降低距离感，但频率很低，点到为止

## Phase 4: 审稿与打磨

初稿完成后：

1. **简介引言审查** — 简介引言是否准确概括了文章的中心思考？和正文立场是否一致？是否信息密度够高、不空洞？
2. **删装饰性修辞** — "聪明但不增加信息量"的句子砍掉。比喻如果只是为了好看而不是帮助理解，直接删。信任读者自己得出结论，不把话说透
2. **加判断锐度** — 模糊表述换成更尖锐的判断。加入更精确的概念，加英文标注给需要的读者
3. **信息密度检查** — 同样字数里能不能多塞一个事实或概念？宁可多一个信息点，也不多一句情感渲染
4. **硬性规则核查** — 逐条检查：没有 emoji，没有感叹号，没有祈使句，没有破折号，没有多余的过渡语气词，**严禁所有"不是X...是Y"句型变体**（包括：不是X，而是Y / 不是X，是Y / 不只是X，更是Y / 不是X而是Y）——无论是否有逗号，无论X和Y多短，一律重写为正向陈述
5. **中英文用词审查** — 扫描全文所有英文词汇，输出一张对照表给 Ody 确认后再改：
   - **保留英文**列：真正的专业术语（如 Agent、LLM、CI/CD、orchestrator、sub-agent、token、reward shaping 等），首次出现加中文标注
   - **改为中文**列：有自然中文对应的非专业词汇（如 session → 会话、spawn → 启动、issue → 任务、PM → 项目经理、input/output → 输入/输出）
   - 判断标准：如果一个词在中文技术社区已经是通用说法（如 Agent、token、prompt），保留；如果只是英文日常词汇混入中文句子，换成中文
   - **等 Ody 确认对照表后再统一修改**，不要自行决定
6. **事实核查** — 数据准不准？案例对不对？上网再确认一遍 Phase 2 中没核实的内容
6. **配图检查** — 配图标注到位了吗？封面方向定了没？参考图链接附上了吗？
7. **平台合规** — 检查绝对化用语（"最好""第一""100%"）和敏感词，避免被限流
8. **标签规划** — 准备 5-10 个相关话题标签（#xxx），混搭大流量标签和精准长尾标签
