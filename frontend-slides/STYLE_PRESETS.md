# Style Presets Reference

Curated visual styles for Frontend Slides. Each preset is inspired by real design references — no generic "AI slop" aesthetics. **Abstract shapes only — no illustrations.**

**Viewport CSS:** For mandatory base styles, see [viewport-base.css](viewport-base.css). Include in every presentation.

**Vibe:** Confident, bold, modern, high-impact

**Layout:** Colored card on dark gradient. Number top-left, navigation top-right, title bottom-left.

**Typography:**
- Display: `Archivo Black` (900)
- Body: `Space Grotesk` (400/500)

**Colors:**
```css

    --bg-primary: #1a1a1a;
    --bg-gradient: linear-gradient(135deg, #1a1a1a 0%, #2d2d2d 50%, #1a1a1a 100%);
    --card-bg: #FF5722;
    --text-primary: #ffffff;
    --text-on-card: #1a1a1a;

```

**Signature Elements:**
- Bold colored card as focal point (orange, coral, or vibrant accent)
- Large section numbers (01, 02, etc.)
- Navigation breadcrumbs with active/inactive opacity states
- Grid-based layout for precise alignment

### 13. Notion Editorial

**Vibe:** Friendly, organized, trustworthy, document-like — like a well-curated knowledge page that earned the audience's attention by feeling familiar

**Layout:** White "page card" floating on warm gray shell. Each slide is structured like a Notion page: breadcrumb header + slide number top, large emoji "page icon" above title, content body, mono-font footer attribution. Tactile, structured, never showy.

**Typography:**
- Display: `Cabinet Grotesk` (700/800) — distinctive but readable, warmer than Inter
- Body: `General Sans` (400/500) — clean alternative to Inter, geometric but warm
- Mono: `JetBrains Mono` (400) — for footer labels, slide numbers, kbd hints

**Colors:**
```css
:root {
    /* Notion's actual surface palette — warm off-whites */
    --bg-shell: #e9e9e7;       /* Outer app shell */
    --bg-page: #ffffff;         /* Main page card */
    --bg-page-warm: #fbfbfa;    /* Warm subtle inset */

    /* Notion's actual text colors — warm dark, NOT pure black */
    --text-primary: #37352f;
    --text-secondary: #787774;
    --text-tertiary: #9b9a97;

    /* Borders & dividers */
    --border: #ebebea;
    --border-strong: #d3d2cf;

    /* Notion's signature muted accent palette (callout-friendly) */
    --red: #e03e3e;     --red-bg: #fdebec;
    --blue: #0b6e99;    --blue-bg: #ddebf1;
    --orange: #d9730d;  --orange-bg: #faebdd;
    --green: #0f7b6c;   --green-bg: #ddedea;
    --yellow: #cb912f;  --yellow-bg: #fbf3db;
    --purple: #6940a5;  --purple-bg: #eae4f2;
}
```

**Signature Elements:**
- **Page card metaphor:** every slide is a white card with subtle shadow on warm gray shell. Border-radius ~12px. Box-shadow with multiple layers for depth without drama.
- **Page header bar:** breadcrumb (icon + path text like `/ Hi, I'm Ody`) on left, slide number (`02 / 09`) in mono font on right, divider underneath.
- **Page icon:** large emoji (~3.75rem) sits above the title, like Notion's page icons. Sets the "voice" of each slide.
- **Callout boxes:** Notion's signature element. Tinted background (yellow/red/blue/green/orange/purple `_bg` color) + emoji icon + text. Use for the "key insight" of each slide. Strong narrative device.
- **Todo lists:** square checkbox (1.5px border, 3px radius) + text. Excellent for "self-check" content where audience reads silently and self-identifies.
- **Card grids:** 2-up or 3-up cards with warm white background, subtle borders, emoji icons, h3 title, body text. Tinted variants for emphasis (purple/blue/green backgrounds).
- **Footer attribution bar:** mono-font labels on left/right, divider above. Subtle, document-like.
- **All accents are MUTED.** Notion's palette is intentionally restrained — never neon, never saturated. The aesthetic comes from structure, not color.
- **Emoji used as content semantically** (page icons, callout markers, list bullets), not decoratively. This is core to the Notion vibe and the only context where emoji is appropriate by default.
- **Hypothesis / pull-quote slides:** center-aligned, large display font, single italic accent word in `--red` for emphasis. Generous whitespace, no decoration.

**Best for:** Validation pitches, founder talks, internal product reviews, anywhere trust and clarity matter more than visual flash. The familiar Notion aesthetic disarms skepticism — it feels like a document, not a sales deck.

## DO NOT USE (Generic AI Patterns)

**Fonts:** Inter, Roboto, Arial, system fonts as display

**Colors:** `#6366f1` (generic indigo), purple gradients on white

**Layouts:** Everything centered, generic hero sections, identical card grids

**Decorations:** Realistic illustrations, gratuitous glassmorphism, drop shadows without purpose

## CSS Gotchas

### Negating CSS Functions

**WRONG — silently ignored by browsers (no console error):**
```css
right: -clamp(28px, 3.5vw, 44px);   /* Browser ignores this */
margin-left: -min(10vw, 100px);      /* Browser ignores this */
```

**CORRECT — wrap in `calc()`:**
```css
right: calc(-1 * clamp(28px, 3.5vw, 44px));  /* Works */
margin-left: calc(-1 * min(10vw, 100px));     /* Works */
```

CSS does not allow a leading `-` before function names. The browser silently discards the entire declaration — no error, the element just appears in the wrong position. **Always use `calc(-1 * ...)` to negate CSS function values.**
