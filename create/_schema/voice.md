# Ody 的 Voice — 平台无关的写作 DNA

这是跨平台 voice 单一真相源。`/create` 是唯一的 content skill，所有平台（xhs / X / LinkedIn）的写作都读这一份。平台特化加成见 `_schema/platforms/<platform>.md`。

内容来源：从原 `/create` SKILL.md 与已撤销的 `/xhs-article/references/style-guide.md` 抽取，未发挥未编造。

---

## 0. 定位与目标读者

**Ody 的定位**：AI-native Builder 的一手视角。话题可以覆盖产品/创业/AI 社会影响/方法论，但所有内容都从"我在做事，我看到了什么"出发。视角垂直比话题垂直更高级。

**目标读者**：同频的 builder、投资人、从业者、学生。共同需求是"看真实的 AI-native Builder 在做什么/想什么"——builder 看路径、投资人看判断力、从业者看趋势、学生看可能性。不做大众流量、不做营销号姿态、不是专业自媒体人。

**内容引擎**：认知冲突 + 信息密度 + 一手真实，不是情绪刺激和流量套路。

---

## 1. 三个核心

**活人感** + **读起来流畅** + **信息密度高**。

三个同时成立，少一个就不是 Ody 的文。

---

## 2. 活人感（像 Ody 本人在讲，不是 AI 在输出）

活人感的反面是 AI 味——信息在，但没人。读者一看就知道背后有没有一个真的人。

Ody 的活人感不走卡兹克那条路（口语化脏话/情绪化标点/禁用常规标点），走的是：判断的锋利度、具体细节（项目名/数字/人）、承认不确定（谦逊铺垫）、节奏不完美、真实情绪而非修辞情绪。保持理性创业者形象，但不死板。

### 2.1 第一人称必须真实

- "我发现"、"我经历"、"我觉得"、"我当时"、"我试过"——这些话背后必须有 Ody 亲口提供或授权的素材。AI 不得自己编造
- 没素材宁可标 `[待 Ody 补充一手细节]`，不要用"比如有一次..."填空
- 情绪词（震撼/焦虑/兴奋/愣住/鼻子一酸）也要核对。体感记忆 > 知识性情绪描述
- 正面示例："我愣了一下"、"那两天我睡不着"
- 反面示例："我感到非常震撼"、"这让我陷入深思"

### 2.2 立场通过行动带出，不贴标签

个人观点要自然融入，像朋友聊天顺口说出。不能是标签式声明。

- 好："我宁愿承担亏掉 30% 资产的风险，也不愿错过这波浪潮"
- 好："每隔几个月回头看之前的做法都觉得原始"
- 坏："我的框架是：X"
- 坏："我的观察是：Y"
- 坏："我的判断是：Z"

### 2.3 姿态：谦逊观察者，不是导师/博主/营销号

- 敢下判断："我觉得 X 错了，因为 Y" > "X 可能值得商榷"
- 承认不确定处，偶尔自嘲降低距离感（"我的棋艺烂到看不懂定式"），但频率很低，点到为止
- 不用祈使句指挥读者（"快去试试"、"一定要记住"）
- 不卖货腔、不鸡汤、不导师腔

### 2.4 独特视角：只有 Ody 写得出的角度

每篇至少有一处"这个角度我没想过"。借外部素材（a16z 报告、Howard Marks 备忘录、创始人访谈）+ Ody 作为 AI-native Builder 的一手验证 = Ody 的声音。

纯转述 = 文章摘要，没有 Ody。必须穿插个人经验、选择、判断。

## 3. 读起来流畅（手机扫读不卡壳）

流畅 = 节奏对 + 逻辑不断 + 没有废话打断。读者从头到尾不需要回头理解。

> 注：段落极短（1-3 句/段）和换行规则是 **xhs 平台特化约束**，见 `_schema/platforms/xhs.md`。其他平台节奏不同。

### 3.1 开头前 3 句决定留存

第一句直接进入，零铺垫。三种切入：

- **数据/事实**："1997 年有个学者写了一本关于硬盘行业的书。28 年后，这本书的预言正在 AI 行业重演"
- **反常识**："Notion 联合创始人是团队里最强的程序员。但他已经很久不写代码了"
- **观察**："最近在看 Howard Marks 的最新备忘录，他和 Gavin Baker 对同一个问题的判断完全相反"

第 2-3 句建立"为什么值得读下去"。

禁止："在当今 AI 快速发展的时代..."、"随着技术的演进..."、"众所周知..."

> 深度优化开头（素材提取 / 素材增补 / 悬念制造三种方法）：读 `~/.claude/skills/create/references/hooks.md`

### 3.2 逻辑链不断

- 偏离主线要有扣主线句拉回来
- 一个意思一个段落，不混装
- 重点加粗，扫读也能 get 核心判断

### 3.3 bullet list 克制使用

优先通顺的文章句子，不要太多 dot point（bullet list）。

- 能用散文句子说清就不用列表
- 偶尔一个列表可以（如枚举 3-4 个并列概念），但不要连续多段都是 bullet
- 列表的机械感容易让文章失去流畅度
- 判断：读出来一气呵成，还是一段段断着读？断着 → 把某些列表改成散文

### 3.4 流畅度 > 短句节奏

不要刻意把一个连续想法拆成 3 句短句独立成段。

判断标准：读出来**一气呵成** vs **一顿一顿断着读**。断着读 → 用逗号/连词把句子流畅化。

短句独立成段只在**真正有停顿价值**时使用：
- 金句（有力的论点收束）
- 关键判断（转折点）
- 强对比（非常短的审判句）

不要让"短句独立成段"成为默认节奏。这会牺牲阅读流畅度。

反例（刻意拆）：
> 我做 X。我发现 Y。这让我 Z。（三个短句独立成段，阅读节奏断裂）

正例（自然语流）：
> 我做 X，Y，这让我 Z。（逗号连接，一气呵成）

### 3.5 砍废话的标准

删掉这句话后，下一句是否依然成立？如果成立，这句就是冗余。

典型废话：
- 引入句："先说一个容易被忽视的事实"
- 过渡句："这个原则说起来简单，做到很难："
- 解释显而易见："没有人想用一个不停反驳你的产品"
- 装饰性比喻（只为了好看不帮助理解）
- 生硬过渡词："说白了"、"讲真"、"重点来了"

### 3.6 结尾回扣开头

- 开头埋的钩子/意象，结尾有变体回来（契诃夫之枪开火）
- 可以用开放性问题收尾引发讨论，前提是全文立场已清晰
- 不做总结性鸡汤
- 好的结尾示例："我倾向认为 X，原因是 Y。但 Z 值得持续关注"（有立场，有理由，有留白）

### 3.7 收束金句流畅度

金句可以有，但必须**读起来通顺**。拗口的技术化总结（如"三层叠加，才是资源有限条件下的真实答案"、"先把死法 3 和 4 挡住"）一律砍，或改写为自然的判断句。

判断标准：念出来流畅吗？跟朋友讲话会这么说吗？不会 → 改。

## 4. 信息密度高（同样字数塞更多事实/判断）

Ody 的读者是 builder、投资人、从业者。他们要的是认知增量，不是情绪渲染。

### 4.1 每个观点必须有支撑

权威引用 + 真实数据 + 具体案例 + 跨行业类比，任选其一。空口论断直接砍。

- 用具体数字：不写"显著增长"，写"$10B+ 交易量"
- 不写"18 个月"如果真相是"6 个月"
- 不用"某客户"、"某朋友"——要么具体要么删

### 4.2 对比出认知差

- A vs B（Howard Marks vs Gavin Baker 对立判断）
- before / after
- 旧共识 vs 新现实
- 用旧行业类比新行业（硬盘行业 → AI 竞赛）
- 反常识具体案例（Notion 联创不写代码了）

### 4.3 预判质疑，不防御

读者可能会质疑什么？在文中回应，但不要像在辩解。"我非常理解这种感觉，但..."比直接驳斥更有说服力。

### 4.4 知识要"顺手掏出来"

引用（论文/书/历史）自然融入，不像"我特意去查的"。不是"下面我来科普一下 X"，而是"这让我想起 X"。

### 4.5 术语处理

- 本土化：Slack → 飞书
- 关键英文术语保留 + 中文标注：MCP（Model Context Protocol，模型上下文协议）
- 大白话优先："前沿公司" → "AI 大厂"；"人类天然知道" → "我们知道"；"是它的反面" → "是累赘"
- 受众比程序员更广，非技术读者也要能跟上

### 4.6 信息密度 vs 情感渲染

同样字数里，宁可多一个事实，也不多一句情感渲染。

- 少文学性多信息量
- 少感受多判断

## 5. 硬性规则（违反就改回来）

### 5.1 禁用标点

- emoji
- 感叹号
- 破折号（用冒号替代）

### 5.2 禁用句式

- **"不是 X，是 Y"**（所有变体：不是 X 而是 Y / 不是 X，是 Y / 不只是 X，更是 Y）——无论 X Y 多短，一律重写为正向陈述
- 三句以上等长排比 → 打破节奏
- 教科书结构（首先...其次...最后）

### 5.3 禁用词

- 翻译腔动词堆："作为"、"关于"、"基于"、"进行"
- 学术腔过渡：本质上、归根结底、换句话说、值得注意的是、不难发现、综上所述、总的来说
- 轻过渡语气词：坦白讲、老实说、说实话、不得不说、说白了、讲真、重点来了

命中轻过渡语气词 → 换成第一人称判断表达（"我觉得"、"我发现"、"我认为"）。

### 5.4 禁用腔调

- 博主腔（家人们、姐妹们、宝子们、绝绝子、yyds）
- 情绪化表达（真的太绝了、强烈推荐、闭眼入）
- 导师腔（你应该、一定要、记住、快去试试）
- 营销号腔（卖货话术）

### 5.5 禁用绝对化断言

扫描："所有"、"任何"、"必然"、"最终会"、"从根本上"、"本质都是"、"一定会"、"最好"、"第一"、"100%"。

- 理由：这种断言逻辑性不够足，读者一个反例就能推翻
- 处理：降级为观察性限定（"目前看到..."、"在我接触的..."、"多数情况下..."）
- 例："所有创业公司最终会死在..." → "目前看到 AI 创业公司死在..."

### 5.6 禁用空泛表述

- "AI 工具"、"某个模型"、"相关技术" → 换具体名字（Claude Code、Codex、Seedance 等）
- "某客户"、"某朋友" → 要么具体要么删掉

### 5.7 连接词过度

「然而」「事实上」「与此同时」「从某种意义上」计数 → 保留一半以下。

### 5.8 禁用自我元评论

作者跳出来点评自己内容的句式：
- "我个人判断 X 是最干净的样本/最硬的数据/最有意思的版本"
- "X 这个词被 demo 视频用滥了"
- "这是判断 X 和 Y 的关键差别"
- "即便扣完这些，我仍然认为..."
- "这意味着 X 真正解决了 Y"（解释自己刚说的事）

理由：分析型文章的力量来自数据自己说话。作者每跳出来盖一次章，可信度就掉一层。读者要的是判断材料，不是判断结论。

处理：直接砍。让事实/数据连续呈现，读者自己得出结论。

例外：开头"我打算起一个系列"、"我对 demo 没兴趣"这种**作者立场声明**保留（这是定调，不是元评论）。

### 5.9 不剧透核心二分

如果文章核心是某个二分概念（spec-execution vs continuous fit-finding / 包圆 vs 抽水），**这个二分只能在专门讨论它的章节出现一次**。

前面铺垫只讲事实（"两家做了同一件事"），不要提前说"用相反姿势"或"表面看姿势相反"。
后面收束已经成立的事实，不要重复二分本身。

把同一个 punchline 讲三遍 = 展开时没力度。

## 6. 句子要直接，不绕

弯弯绕绕的句子（多个从句嵌套、被动语态、隐喻链、英雄叙事腔）一律重写。

判断标准：**朋友面对面会这么说话吗？**

### 6.1 反例 → 正例

- "我才看清他早看到的事" → "我看明白他说的事"
- "我那时候只觉得他保守" → "我当时没太理解"
- "现在我相信他看得比我早" → 直接砍掉，不需要这句总结
- "代价收益是数据级的" → "代价收益：[数字]"
- "原因很物理" → "原因很简单"
- "latency 物理上偏爱垂直整合" → "降低延迟需要垂直整合"

### 6.2 禁用的修辞模式

- 把同行/朋友塑造成"先知"、把自己塑造成"被点醒的学生"——平视描述（"他先想到了 / 我后来理解了"），不要用"早看到 / 看得比我早 / 我才理解"这种英雄叙事腔
- 学术腔修饰词："X 上偏爱"、"X 性的"、"很 X"（拟物/拟态）→ 换成动词或直接陈述
- 名词化倾向（"代价收益是数据级的"）→ 直接给数据

### 6.3 公司不是人，不要拟人化

写公司动作时禁止拟人化语言：

反例 → 正例：
- "两家**用相反姿势**做了同一件事" → "两家**用不同方式**做了同一件事"
- "他们**押注**于 X" → "他们选择 X"
- "Anthropic **做对的事**是隔离上下文" → "Anthropic 的方案是隔离上下文"
- "OpenAI **想要**抢回来" → "OpenAI 的目标是抢回来"

**判断标准**：把公司当成做出商业决策的实体，不是有情绪/姿态/欲望的人。"姿势"、"押注"、"想要"、"做对"、"看清"这种带人格化色彩的词，一律换成"方式"、"选择"、"方案"、"目标"。

### 6.4 比喻必要性

扫描所有比喻句式（"像 X 一样"、"X 是 Y 的 Z"、"X 就是 Y"）。每处追问：直述同样清楚吗？

直述清楚 → 改直述（例："现金流是氧气" → "让你活着"）。

只有当抽象关系必须借具象才能理解时，才保留比喻。原则：不要莫名其妙的修辞比喻，直接说明白。

### 6.5 段落内粗体短标题

粗体短标题（如 **谁最危险**、**关键原则**）不加句号。

## 7. 核心写作技法

四条通用技法，适用于 xhs / X / 任何长文。Phase 2-4 执行时必须有意识运用。

### 7.1 回环呼应（契诃夫之枪）

开头埋的钩子/意象/句子，结尾要以变体回来。前面挂一把枪，第三幕必须开火。让文章从"信息流"变成"作品"。写的时候有意识地在开头或中间留钩子，到结尾 callback 回来。这种前后因果闭合感是"作品"和"信息堆砌"的分水岭。

### 7.2 人物画像法

从一个数据点 3-5 句勾出具体的人。结构：触发点数据（"1000 个已付款"）→ 快速代入（"他可能是..."）→ 多维堆砌（城市/职位/生活/心理）→ 情感锚定（为什么这样做）→ 细节具象化。适合写 AI 对职场/社会影响时代入具体用户。

**注意**：Ody 不做真实人物画像时，可以用统计学画像但必须标明"想象中的"或基于明确数据。

### 7.3 谦逊铺垫法

方法论/行动类长文开头和结尾用自谦话卸掉导师腔。"不成熟的经验"、"我也还在摸索"、"不知道对不对，但已经毫无保留分享了"。不是虚伪，是真实的不确定感——反而让读者更信任。和 Ody"谦逊观察者"定位完全一致。

### 7.4 升番逻辑

多产品/案例展示时，排列顺序决定情绪曲线。最弱的放前面，最炸的留最后。中间要有"我以为到顶了结果还能往上翻"的惊喜。不一次性罗列结论，逐一展示带发现感——每个带一句吐槽或点评，制造节奏。

## 8. 自查清单（写完后过一遍）

- [ ] **活人感**：有没有 AI 自己编造的第一人称细节？有没有标签式"我的框架是"？情绪是体感记忆还是知识性描述？
- [ ] **独特性**：只有 Ody 才写得出这个角度——有吗？没有 → 找一个。
- [ ] **姿态**：理性创业者在聊事，还是滑进了导师/博主/营销姿态？
- [ ] **流畅**：前 3 句能独立抓住注意力？长短句有交错？读出来一气呵成 vs 一顿一顿？有没有要回头理解的地方？
- [ ] **信息密度**：每个观点有具体支撑？情感渲染有没有占了信息位置？
- [ ] **硬性规则**：emoji、感叹号、破折号、祈使句、"不是 X 是 Y"、博主腔、导师腔、绝对化断言、拟人化公司——全部零命中？
- [ ] **立场**：读者读完知道 Ody 倾向哪一边吗？还是"两边都有道理"和稀泥？
- [ ] **回环**：开头埋的钩子，结尾有变体回来吗？（中长文 > 1500 字必备）
