## 6 原型 + 写法重心 + 核心技法

| 原型 | 典型场景 | 写法重心 | 核心技法 |
|------|---------|---------|---------|
| **实战复盘型** | 做了具体项目/咨询/产品试错（Cliff 咨询、SodaBOS、RayFeedback） | 过程叙事 + 一手发现 + 坦诚的坑 | 英雄之旅弧（困境→尝试→发现→变化）+ 回环呼应 |
| **产品思考型** | 用 Claude/某产品有深度感触 | 场景 + 独立判断 + 同类对比 | 升番逻辑（多案例展示）+ 回环呼应 |
| **思考笔记型** | 趋势观察 + 创业感想 + 心态/哲学随笔 | 现象/心态 + 独立判断 + 更自由的形式 | 人物画像法（代入具体读者）+ 文化升维 |
| **方法论分享型** | 沉淀的做事方法/工具用法/心得 | 每节落到可执行动作 + 坦白学习曲线和失败点 | 谦逊铺垫法（开头和结尾）+ 回环呼应 |
| **技术博客分析型** | 拆解 Anthropic / OpenAI / 大厂技术博客，给独立判断（不是翻译式总结） | 拆方案 → 找隐藏假设 → 命名概念二分 → 对照自己的工作流 → 拆开取用（哪部分用、哪部分不用） | 概念二分（spec-execution vs continuous fit-finding 这种核心二元命名）+ 隐藏假设盘点 + 回环呼应。**禁止博眼球标题**——技术读者反感营销腔，标题用事实陈述+概念二分，避免感叹/反转钩子 |
| **AI 产品落地分析型** | 拆解一家垂直 B2B AI 公司（Heidi、Harvey、Klarna、Decagon、Mercor、EvenUp 等），用硬数据回答"这玩意到底干没干活" | 公司从何而来 → 给谁用 → 解决什么 → 痛点二级证据 → 怎么解决 → ROI 是多少（带源） → 钱与规模 → 与同类对比 → 风险与局限 → 收束回环 | **必备 10 段骨架**（见下）+ ROI 升番（个人→机构→国家/全球级）+ 数据全部具名（人/机构/时间/数字缺一不可）+ 同赛道对照（至少 2 个竞品具名带数字）+ 诚实降级（必有"我得说几个不好的地方"段）+ 回环呼应作为独立收束段。**门槛硬性**：缺任一必备段（尤其是 ROI 数字 + 估值/融资 + 风险段）= 不写此原型，改回思考笔记型 |

## AI 产品落地分析型 — 必备 10 段骨架

新原型门槛硬性，结构不能省。Phase 2 大纲必须按这 10 段铺，缺必备段就在大纲里标 `[缺]` 让 Ody 决定补还是降级原型。

| # | 段名 | 必须包含 | 反例（这样写就是降级） |
|---|------|---------|---------------------|
| 1 | **钩子 + 创始人故事** | 具体场景切入 + 创始人具名 + 为什么做（个人 burnout / 行业痛点 / 技术拐点，必须有一个具体动因） | "AI 改变了 X 行业..." 这种宏大开头 |
| 2 | **给谁用（客户画像）** | 具体行业 + 具体岗位 + 客户规模数字（多少医生/律师/客服在用，带时间） | "面向企业用户" / "服务专业人士" |
| 3 | **解决什么问题** | 量化的痛点（医生每天文书 X 小时 / 律师合同审 Y 天 / 客服解决率 Z%）+ 行业级第三方数据（BMA / Gartner / 行业协会） | "提升效率"、"降本增效" |
| 4 | **痛点二级证据**（强烈推荐，分析型的杀手段） | AI 出现之前这个痛点已经被市场用什么方式解决了？花了多少钱？这把"问题主观重要性" → "市场已验证的经济事实"。例：Heidi 段写美国医生雇印度远程 scribe 1500-3000 美元/月，上亿美元外包行业 | 跳过此段 = 失去最强的"问题严重性"证据 |
| 5 | **怎么解决（产品逻辑）** | 关键产品决策（开发流程 / 模型策略 / 集成方式 / 数据隐私设计）+ 至少 1 处独立判断 | 翻译产品官网 features list |
| 6 | **ROI 升番（核心）** | 至少 3 层：个人级（一个用户省 X 小时）→ 机构级（一家诊所/事务所减 Y%）→ 国家或全球级（NHS / 数万家所 / 数百万次）。每个数字带人/机构/时间/来源 | 单点数据 + 模糊大词（"显著提升"） |
| 7 | **钱与规模** | 估值（带轮次和日期）+ 融资额 + 投资方至少 2 家具名 + 营收 / ARR（如有公开） | "估值数亿"、"已完成多轮融资" |
| 8 | **同赛道对照** | 至少 2 个竞品具名 + 各自的差异化数据（定价 / 客户数 / 技术路线）+ 这家凭什么赢 | 只夸主角不提竞品 / 提了竞品但无数据 |
| 9 | **风险与局限** | 至少 3 个具体风险：方法学（RCT / 临床证据缺失）+ 场景边界（哪类用例还干不好）+ 监管/隐私/合规 | "也存在一些挑战" / 完全跳过此段 |
| 10 | **收束 + 回环呼应**（独立成段，不是结尾点缀） | 把第 1 段创始人故事的钩子（人名 / 离开医院的瞬间 / 一句金句）callback 回来。读完一堆数据后，读者需要一句金句把人物和数据收回来 | 直接结束 / 用"总结一下..."这种总结腔 |

**ROI 一句话原则**：每个 ROI 数字必须能拆成 `[谁]在[时间]通过[做什么]让[什么指标]变化了[多少]`。凑不出这个句子的数字 = 不写。

**段 4「痛点二级证据」追问清单**（写其他赛道时复用）：
- 法律 AI（Harvey）：律所之前合同审是怎么解决的？外包多少钱？律师工时多贵？
- 客服 AI（Decagon）：之前解决率怎么提的？BPO 海外外包每席位多少？人工解决一票多少钱？
- 销售 AI（Clay）：SDR 工资 + 工具栈成本是多少？外包 SDR 服务多少钱/月？
- 招聘 AI（Mercor）：HR 筛简历多少小时？猎头费率？背调外包成本？
- 这一段写不出 = 你对这个行业的痛点理解还不够深，先不要写此原型

## 配图规则（分析型默认包含 3 张图，覆盖普适配图禁令）

普适配图规则的「默认不配」对分析型不适用。分析型整篇都是数据 + 抽象商业逻辑，纯文字会让密集信息变成劝退墙。Heidi 篇成功的一个关键正是 3 张外部图（创始人头像 + 产品 UI + 团队照）把 14 页的密度切成可呼吸的节奏。Harvey 篇缺这 3 张，文字版正确但视觉上比 Heidi 篇枯燥。

分析型默认配 **3 张外部图**，每张承担明确的角色：

| 位置 | 图类型 | 来源 | 作用 |
|------|--------|------|------|
| **段 1（钩子页）** | 创始人 / 主角人物像 | TechCrunch profile / Bloomberg / 创始人个人网站 / LinkedIn | 给一个"人"，把抽象公司变成具体个体；让读者代入故事 |
| **段 5 或段 6（产品页）** | 产品 UI 截图 / 产品演示图 / 架构图 | 公司官网 / 公司 blog / 产品 marketing 页 | 把抽象的"产品分四块"变成"长这样"；分析型最容易过抽象，UI 图是解药 |
| **段 10（收束页）** | 团队合影 / 办公室 / 主角近照 | 公司官网 about 页 / 创始人 X 自发 / 媒体专访 | 收束的情绪锚点；让数据讨论有人味结尾 |

**Phase 2 大纲必须执行**：
1. 列 3 张图的具体位置 + 描述 + 来源 URL
2. 立刻下载图到 `/Users/odyzhou/Desktop/xhs-build/[slug]/images/` 目录
3. 检查图分辨率 ≥ 1840px 宽（Phase 6 在 2x DPI 下渲染需要）
4. 如果某个角色找不到合适图（例如产品 UI 没公开截图），在大纲里标 `[图缺：原因]` 让 Ody 决定补图还是裸跑

**例外**：如果是纯财务/数据型分析（比如纯讲财报、估值倍数对比），可以走信息图替代外部图。但创始人故事 + 产品形态两段几乎永远值得配真图。

**反例（Harvey 篇的教训）**：把整篇分析当成纯思考笔记型来配图（默认不配），结果 11 页全是文字 + 表格 + 卡片。视觉上单调，跟 Heidi 篇并排看一眼就知道差在哪。下次写分析型，**Phase 2 大纲第一句话先列 3 张图的位置和来源 URL**，不要等 Phase 6 才想起这件事。
